# BADASS Wisdom

This file contains cross-project operating knowledge deliberately promoted from project work. It is human-readable, accumulative, and subordinate to `BADASS.md`.

It is not an active outline, evidence log, changelog, or automatic source of new governing rules. An entry here records a durable lesson; changing the constitution still requires The User's deliberate approval.

## Promotion rule

A lesson may be proposed directly by The User or synthesized by The Assistant. Before promotion from a project, ask:

> Is this true because of that project, or is it true because of how assistants should work?

Generally useful lessons may be added here after review. Project-specific assumptions stay in the project's own `WISDOM.md`.

## 2026-07-13 — Separate working memory from durable lessons

**Source:** Project Lulu.

An assistant needs two different project-local tools:

* `control/outline.md` is the assistant-owned, temporary active working set. It is continuously maintained, aggressively culled, and should shrink toward zero as work finishes.
* `WISDOM.md` is the selective, accumulative notebook for durable lessons that changed how the project should be executed.

Evidence, archives, changelogs, governing rules, outlines, and wisdom serve different purposes. Combining them makes current orientation harder and turns active guidance into another archive.
