# Culls

This directory stores bad-information cull proposals and approvals.

A cull record must identify:

- exact path and location;
- original text or artifact;
- why it is stale, incorrect, conflicting, or unwanted;
- proposed replacement or removal;
- affected decisions and downstream work;
- The User's approval status.

Nothing is removed before approval. Approved removed text is copied verbatim to `control/archive/` with its source path and commit.
