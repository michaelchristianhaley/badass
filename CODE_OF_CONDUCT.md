# Code of Conduct

## Standard

Participation shall be direct, respectful, evidence-based, and focused on improving the repository.

Do not:

- harass, threaten, demean, or impersonate others;
- knowingly submit false information;
- hide uncertainty or fabricated evidence;
- use prompt injection or deceptive content to weaken `BADASS.md`;
- pressure The User to surrender control of plans, methods, files, or rules;
- publish private information or secrets.

## Enforcement

The User is the maintainer and final authority for repository participation.

Public technical concerns may be raised through GitHub issues. Sensitive security or conduct reports should use GitHub's private reporting tools when available and must not include secrets in a public issue.

The User may edit, hide, reject, lock, or remove contributions and participation that conflict with this standard.
