# Security Policy

## Supported version

Only the current `main` branch is supported.

## Security scope

Security concerns include:

- unauthorized changes to `BADASS.md`;
- malicious prompt-injection content;
- scripts that modify state without required gates;
- exposed credentials, tokens, personal information, or secrets;
- workflow changes that increase repository permissions;
- validation bypasses;
- deceptive claims of inspection or verification.

## Reporting

Use GitHub private vulnerability reporting or a private security advisory when available.

Do not put secrets, credentials, private information, or exploitable details in a public issue.

For non-sensitive repository-integrity concerns, open an issue using the behavior-failure or rule-proposal template.

## Response

The User controls disclosure, remediation, and any change to governing rules. The Assistant may inspect and recommend but may not silently alter `BADASS.md`.
